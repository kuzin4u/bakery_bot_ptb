# 🍞 Telegram-бот для пекарни Massa Madre

Бот для продажи хлеба на закваске, зефира, творожной массы и других продуктов пекарни.  
Реализован на **python-telegram-bot** + **Starlette** + **Uvicorn** с поддержкой вебхуков для продакшена.

---

## 📦 Функционал

- 🍞 **Каталог товаров** – просмотр продуктов с описанием и ценой.
- 🧺 **Корзина** – добавление/удаление товаров, просмотр итоговой суммы.
- 📦 **Оформление заказа** – сбор адреса и комментария, создание заказа в БД.
- 📋 **История заказов** – просмотр своих заказов с деталями.
- 👤 **Профиль** – просмотр данных пользователя (имя, телефон, адрес).
- 🤖 **AI-консультант** – (заглушка) возможность задать вопрос о продукции.
- 💾 **Сохранение данных** – пользователи, товары, заказы хранятся в SQLite.
- 🖨️ **Подготовка к печати** – (можно добавить позже).

---

## ⚙️ Технологии

- **Python 3.10+**
- **python-telegram-bot** (v20.7) – работа с Telegram API
- **Starlette** (v0.37) – ASGI-фреймворк для вебхуков
- **Uvicorn** (v0.30) – ASGI-сервер
- **SQLite** – лёгкая встроенная БД
- **python-dotenv** – управление переменными окружения

## 🚀 Установка и запуск

### 1. Клонируйте репозиторий

```bash
git clone https://github.com/your-username/bakery-bot.git
cd bakery-bot

2. Создайте виртуальное окружение


bash
python -m venv venv
source venv/bin/activate      # для Linux/Mac
venv\Scripts\activate         # для Windows

3. Установите зависимости
bash
pip install -r requirements.txt
4. Создайте файл .env
В корне проекта создайте файл .env и добавьте:

env
BOT_TOKEN=ваш_токен_от_BotFather
WEBHOOK_URL=https://ваш-сервис.onrender.com   # только для продакшена (опционально)
Важно: .env должен быть добавлен в .gitignore, чтобы не хранить секреты в репозитории.

5. Запустите бота
Локально (режим polling):
bash
python bot.py
Бот будет работать без вебхука, используя long polling.

В продакшене (на Render):
В настройках Render укажите команду запуска:

text
python bot.py
А в переменных окружения задайте BOT_TOKEN и WEBHOOK_URL.

🗄️ База данных
Используется SQLite. Таблицы создаются автоматически при первом запуске:

users – пользователи

products – товары (заполняется вручную или через админ-панель)

orders – заказы

order_items – товары в заказах

Пример добавления товара в БД (можно выполнить вручную через sqlite3 или в коде):

sql
INSERT INTO products (name, description, price, category, is_active)
VALUES ('Хлеб на закваске 740 г', 'Пшеничный хлеб на живой закваске Massa Madre. Вес 740 г.', 400, 'хлеб', 1);
🌐 Деплой на Render
Загрузите код в GitHub-репозиторий.

На render.com создайте Web Service.

Подключите репозиторий.

Настройки:

Environment: Python 3

Build Command: pip install -r requirements.txt

Start Command: python bot.py

Добавьте переменные окружения:

BOT_TOKEN – ваш токен

WEBHOOK_URL – URL вашего сервиса (например, https://имя-сервиса.onrender.com)

Нажмите Create Web Service.

Бот будет доступен через вебхук. При каждом обновлении кода в репозитории Render автоматически перезапустит бота.

🧪 Тестирование
Для тестирования можно использовать режим polling без указания WEBHOOK_URL.
Отправьте боту команду /start и проверьте работу всех кнопок.
